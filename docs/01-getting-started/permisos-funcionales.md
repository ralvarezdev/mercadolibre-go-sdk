## Permisos funcionales

Al crear una aplicación, debes configurar los permisos funcionales de tu integración. Estos serán necesarios para la autorización del usuario a su aplicación, otorgando el Grant y permitiendo el correcto uso de nuestra API en su nombre.

Nota:

 Antes de crear una aplicación, asegúrate que la cuenta que estás utilizando es 

la cuenta del propietario

 en la solución que vas a desarrollar, evitando así problemas de transferencia de cuenta en el futuro. Recomendamos que la cuenta se cree bajo una entidad legal. 

Estos son los permisos disponibles para la configuración de tu integración:

Accede a nuestro DevCenter

. 

## Scopes

- **Solo lectura:** permite el uso de métodos API GET HTTPS.
- **Lectura y escritura:** permite el uso de métodos API PUT, POST y DELETE HTTPS.

## Error por falta de Permisos

El error a continuación es retornado caso su integración no tenga acceso al recurso por falta del Permiso funcional correspondiente:

**Respuesta:**

```javascript
{
"code": "PA_UNAUTHORIZED_RESULT_FROM_POLICIES",
"blocked_by": "PolicyAgent",
"message": "At least one policy returned UNAUTHORIZED.",
"status": 403
}
```

Para solucionar deberás acceder las configuraciones de su aplicación y habilitar el Permiso funcional asociado, activando a los escopos de lectura y escritura.
